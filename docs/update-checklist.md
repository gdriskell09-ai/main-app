# Update Checklist

Use this before and after any future implementation session.

## Before Work

- Run `pwd`.
- Run `git status --short`.
- Confirm the repo is `main-site`.
- Read `PROJECT_STATE.md`.
- Read `docs/phase-rules.md`.
- List files you plan to touch.
- Confirm the requested work is not forbidden for now.
- Confirm whether package installs are approved.

## During Work

- Keep changes small.
- Do not mix `scrub-club-app` context.
- Do not hard-code final pricing.
- Do not add paid API dependencies without approval.
- Keep AI button-triggered only.
- Keep blueprint/fallback generation working with zero AI keys.
- Keep roadmap modules marked future unless explicitly approved.

## After Work

- Run the approved build/test command if code changed.
- Report files changed.
- Report build/test result.
- Report manual browser tests.
- Report what remains later.
- Update docs only if requested or if the task was a docs phase.

## Website Quality Checklist

- Business name is visible.
- Industry/service category is clear.
- Service area is clear.
- Main CTA is obvious on desktop and mobile.
- Services are specific and believable.
- Copy does not invent reviews, licenses, awards, or guarantees.
- Pricing is not final unless approved.
- Contact/lead flow is clear.
- Site works without AI.
- Fallback/blueprint content remains valid.
